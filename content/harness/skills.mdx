| Skill | Status | What it does |
| --- | --- | --- |
| `design-consultation` | Exists | Answers design system and brand questions with citations into this standard |
| `design-review` | Exists → upgrading | Lints a screen against the control catalog; findings prioritised P1 (must fix) / P2 (should fix) / P3 (nice to fix) |
| `design-page` | Planned | Orchestrates the six-phase loop; refuses to skip gates |
| `design-standards` | Planned | Catalog filtering, tier application, waiver protocol |
| `content-style` | Planned — V1 priority | Applies voice & tone + naming at generation time, so copy arrives on-voice by default |

Every skill page here will contain: what it does in one sentence · when to reach for it · the exact copy-paste invocation · one worked example (real input → real output) · what good output looks like · what to do when it goes wrong.

One deliberate stance: **unwarranted novelty is flagged as readily as generic output.** A custom pattern where a stack component exists is a finding.
