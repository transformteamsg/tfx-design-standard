| Skill | Status | What it does |
| --- | --- | --- |
| `tfx-design-ui` | Exists | Orchestrates the six-phase loop (intent → diverge → plan → implement → verify → ratchet); refuses to skip gates |
| `tfx-design-standards` | Exists | Catalog reading and filtering, tier application, waiver protocol |
| `tfx-design-review` | Exists | Evaluator procedure for grading a finished design; findings sort by tier into blocking (L0/L1) and advisory (L2). Used by the tfx-design-evaluator subagent, never the generator |
| `tfx-content-style` | Exists | Applies voice & tone, naming, and the AI-writing tells (SLP-9) at generation time, so copy arrives on-voice by default |
| `design-consultation` | Planned | Answers design system and brand questions with citations into this standard |

Every skill page here will contain: what it does in one sentence · when to reach for it · the exact copy-paste invocation · one worked example (real input → real output) · what good output looks like · what to do when it goes wrong.

One deliberate stance: **unwarranted novelty is flagged as readily as generic output.** A custom pattern where a stack component exists is a finding.
